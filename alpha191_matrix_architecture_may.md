## 4. 权重校准方法

### 4.1 方法一：IC-IR加权法（推荐第一版）

$$
w_i = \frac{IC_i \times IR_i}{\sum_{j=1}^n IC_j \times IR_j}
$$

其中 IC_i 取绝对值。若有需要处理负IC因子（反转类），则：

$$
w_i = \frac{|IC_i| \times IR_i \times sign_i}{\sum |IC| \times IR} \quad \text{需归一化到正数}
$$

**优点**：简单、可解释、数据驱动
**缺点**：未考虑因子间相关性，静态权重

### 4.2 方法二：均值-方差优化（MVO）

最大化夏普比：
$$
\max_w \frac{w^T\mu}{\sqrt{w^T\Sigma w}}
$$

其中：
- μ = 各因子的条件期望IC（每情景）
- Σ = 因子协方差矩阵（需要日级IC时间序列）

**需要的数据**：190因子×交易日数的日级IC面板

### 4.3 方法三：滚动回测校准（最务实）

```
针对每个情景：
  1. 固定基础引擎权重（trend×0.381 + struct×0.095 + momentum×0.190 + mf×0.190）
  2. 分配Alpha模块总权重（如 0.30~0.50）
  3. 遍历候选Alpha因子组合的权重组合（网格搜索）
  4. 优化目标：在该情景的历史回测期内最大化 夏普比 / Calmar比
```

#### 权重校准的具体步骤（以S3/chaos为例）

```
基础引擎固定权重：0.70（趋势0.267+结构0.067+动量0.133+资金流0.133）
Alpha模块总权重：0.30

Alpha候选：α052（IC=+0.0357）、α169（IC=+0.0478）、α013（IC=+0.0411）

网格搜索权重空间：
  w_α052 ∈ [0.05, 0.15] step=0.025
  w_α169 ∈ [0.05, 0.15] step=0.025  
  w_α013 ∈ [0.05, 0.15] step=0.025
  约束：w_α052 + w_α169 + w_α013 = 0.30

  总组合：C(5,1)×C(5,1)×C(5,1) ≈ 125种
  回测125次 × 5个情景 = 625次回测
  （大约耗时：625 × 20秒 ≈ 3.5小时，可并行化）
```

### 4.4 推荐：混合方案

```
第一层：IC-IR加权 → 初始权重（理论锚定）
第二层：正交约束调整 → 去重
第三层：滚动回测微调 → 实证校准
```

## 6. 三种可选实施方案

### 方案A：简单替换式（⭐⭐，最小改动）

**思路**：不改框架，只在现有代码中增加情景判断，替换alpha062为不同Alpha

```
在 bt_v13_base.py 中：
  1. 加载 season（已有 get_season()）
  2. 根据季节选择不同的Alpha因子组合
  3. 固定基础引擎权重不变
```

**改动的代码范围**：

```python
# 新增：情景化Alpha选择器
ALPHA_CONFIG = {
    'summer':        {'alpha': 'alpha052', 'weight': 0.143},
    'chaos_spring':  {'alpha': 'alpha052', 'weight': 0.143},  # 与summer共享
    'chaos':         {'alpha': 'alpha169', 'weight': 0.143},
    'chaos_autumn':  {'alpha': 'alpha031', 'weight': 0.143},
    'autumn':        {'alpha': 'alpha062', 'weight': 0.143},  # 回退到默认
}
```

**优点**：
- 只需修改 ~50行代码
- 当天可上线测试
- 几乎无额外计算开销

**缺点**：
- 仅替换一个Alpha因子，效果有限
- 未充分利用多因子组合优势
- 权重未优化（沿用固定143%）

### 方案B：多因子组合式（⭐⭐⭐，推荐第一版）

**思路**：每情景使用2-3个Alpha因子组合，基础引擎权重微调

```
以S3/chaos为例：
  基础引擎 0.65（trend 0.25 + struct 0.06 + momentum 0.12 + mf 0.12 + pos 0.10）
  Alpha模块 0.35（α169 0.15 + α052 0.12 + α013 0.08）
  归一化后 = 1.0
```

**需要的新建数据**：
- 所有候选Alpha因子在 strategy_signal 中的评分值
- 因子协方差矩阵辅助权重决策
- 但不一定需要完整MVO，IC-IR + 正交约束即可

**改动规模**：
- `bt_v13_base.py` 改为 `bt_v13_matrix.py`
- 增加 `load_alpha_bundle()` 批量加载多Alpha
- 增加情景化权重矩阵配置（约200行新代码）
- 增加 `alpha_batch_compute.py` 批量写入各因子评分

**优点**：
- 预期IC提升明显（多因子组合 vs 单因子）
- 权重有数据支撑
- 回测验证可行

**缺点**：
- 需要批量计算所有候选Alpha的评分（可能需要数小时）
- 季节IC的统计稳定性仍需检验

### 方案C：完整矩阵引擎（⭐⭐⭐⭐，长期目标）

**思路**：独立的情景化评分服务，每情景独立训练权重

```
┌─────────────────────────────┐
│       Matrix Scoring Engine │
│                              │
│  ┌─────┐ ┌─────┐ ┌─────┐   │
│  │C1牛 │ │C2震 │ │C3弱 │ → │ (各情景权重不同)
│  └──┬──┘ └──┬──┘ └──┬──┘   │
│     │       │       │       │
│     ▼       ▼       ▼       │
│  ┌──────────────────────┐   │
│  │  Output Normalizer   │   │
│  └──────────────────────┘   │
└─────────────────────────────┘
```

需要的能力：
- 完整因子协方差矩阵（190×190）
- MVO权重优化
- 滚动窗口回测框架
- 情景切换平滑处理（边界效应）
- 动态权重衰减（情景刚切换时的过渡）

**优点**：
- 最完整的Alpha工程实践
- 理论上IC提升最大
- 业界标准做法

**缺点**：
- 开发周期长（2-4周）
- 需要大量数据准备
- 过拟合风险高（情景越多、因子越多、调参空间越大）
- 情景切换边界的跳跃效应需要处理

3. **灵敏度分析**：
   - 权重±5%扰动对最终收益的影响
   - 因子替换的边际影响

## 10. 风险与注意事项

### 10.1 统计风险

```
⚠️ 小样本信用危机：
  summer (108天)：IC看似很强，但10年回测中只出现~3次
  autumn (105天)：有效IC约7天，基本上是"尾部事件"
  
  建议：对小样本情景使用贝叶斯收缩（向全局IC收缩）：
    
    IC_shrunk = (n_season × IC_season + λ × IC_global) / (n_season + λ)
    
    其中 λ 为300（约1.5年，经验值）
```

### 10.2 实现风险

```
1. 因子正交性不足：如果选出的Alpha因子互相高度相关，
   组合效果 ≈ 单因子×3（并没有信息增益）
   
2. 情景误判：season_state的理论标签与实际市场不完全一致，
   如果router判断错误会直接用错Alpha组合
   
3. 计算延迟：多因子评分批量计算可能耗时数小时，
   需要考虑增量更新机制
```

### 10.3 补救方案

```
情景误判后的graceful degradation：
  ┌──────────────────────────────┐
  │ confidence ≥ 0.8 → 使用矩阵  │
  │ 0.5 ≤ confidence < 0.8 →    │
  │   混合（矩阵×0.7 + 基准×0.3）│
  │ confidence < 0.5 → 回退到基准 │
  └──────────────────────────────┘
```

# 评分加载
for r in scores:
    code, td, m1 = r['ts_code'], r['trade_date'], r['score']
    season = get_season(td)       # 用于参数

### proposed: bt_v13_matrix.py

```python
# 配置
MATRIX_CONFIG = {
    'summer': {
        'base_weights': {'trend':0.267, 'struct':0.067, 'momentum':0.133, 'mf':0.133},
        'alpha_weights': {'alpha052':0.15, 'alpha122':0.10, 'alpha093':0.10},
        'alpha_total': 0.35,
        'base_alpha_frac': 0.60,      # 若启用多模块
    },
    'chaos': {
        'base_weights': {'trend':0.267, 'struct':0.067, 'momentum':0.133, 'mf':0.133},
        'alpha_weights': {'alpha169':0.15, 'alpha052':0.10, 'alpha013':0.08},
        'alpha_total': 0.33,
    },
    ...
}

# 评分加载
for r in scores:
    code, td, m1 = r['ts_code'], r['trade_date'], r['score']
    season = get_season(td)
    config = MATRIX_CONFIG.get(season, MATRIX_CONFIG['chaos'])
    
    # 加载情景化Alpha
    alpha_sum = 0
    for af, aw in config['alpha_weights'].items():
        val = alpha_data.get((code, td), {}).get(af, 50)
        alpha_sum += val * aw
    
    # 混合评分
    base_w = 1 - config['alpha_total']
    m1 = m1 * base_w + alpha_sum  # alpha_sum已经做了加权
```
